# 2FA TOTP 共享密钥（用户输入）格式与解码规范

## 摘要

本文档规定：写入本应用并用于 **RFC 6238 TOTP 计算** 的输入，仅为 **`otpauth://totp/...` 中 `secret` 及网站「手动输入密钥」所对应的 RFC 4648 Base32 串**。经 **规范化**（去空白、去连字符）后，按 Base32 解码为密钥字节 $$K$$。**本应用不解析**十六进制密钥、恢复码或其它编码。实现见 `TwoFactorAuth.Core.Totp.SecretBytes`。

## 关键词

TOTP；2FA；共享密钥；Base32；RFC 4648；规范化

## 1. 术语

| 术语 | 含义 |
|------|------|
| **Base32 字母** | RFC 4648：`A`–`Z` 与 `2`–`7`；可选末尾填充 `=`。 |
| **规范化串** | 对用户输入按第 3 节规则处理后的连续字符串，记为 $$T$$。 |

**说明**：服务商另提供的 **账户恢复码**（例如多组独立码）属于 **登录备用** 机制，**不是**本条所述的 TOTP 共享密钥；本应用 **不提供** 恢复码解析，用户亦 **不应** 将恢复码粘贴到密钥字段。

## 2. 排版（非密钥字符）

网站常将 Base32 密钥 **分段** 展示，便于抄写，例如：

`JBSWY-3DPE-HPK3-PXP`

其中的 **连字符** `U+002D`（及同类 Unicode 连字符）**不属于**密钥，第 3 节规范化时 **删除**。

## 3. 规范化算法

**输入**：字符串 $$S$$（可多行）。

**输出**：$$T$$。

对 $$S$$ 中每个字符 $$c$$：若 `char.IsWhiteSpace(c)` 或 $$c \in \{`\u002D`, `\u2013`, `\u2014`, `\u2212`\}$$，则丢弃；否则原样追加到 $$T$$。

不改变字母大小写，不插入字符。

## 4. Base32 解码

1. 若 $$|T| = 0$$：错误码 `empty`。
2. 若 $$T$$ 不满足 **严格 Base32 表面形式**（大写后主体仅含 `A`–`Z`、`2`–`7`，`=` 仅能在末尾连续出现）：错误码 `base32_invalid`。
3. 否则调用 `Base32.Decode` 得 $$K$$；若 $$|K| = 0$$：错误码 `base32_empty`；否则成功。

## 5. 与 TOTP 的关系

$$K$$ 作为 HMAC-SHA1 密钥参与 RFC 6238；位数、步长等见 SRS。
